# AgentArena Backend

Python backend for the IoA dynamic-pricing simulation. Drives the existing
`demo.html` frontend with **real** event streams produced by either rule-based
agents or `claude -p` LLM agents.

## Architecture (the 5-bullet version)

- **Single-agent schema** (`schemas.py::AgentState`) — id, persona (profile +
  description + traits), budget, beliefs, tools, inbox, outbox.
- **Communication matrix** (`schemas.py::CommMatrix`, generated by `topology.py`) —
  an explicit `Dict[node][node] -> bool` that gates every message. **The only
  place topology has any effect** is `matrix[sender][recipient]`. Swap matrix
  → different topology. Agent logic doesn't change.
- **Two actions** (`schemas.py::Action`) — `BUY(target=seller_id)` or
  `COMMUNICATE(target=agent_id, content=text)`. (`WAIT` is the no-op default.)
- **Per-tick decision** — engine calls `agent_runtime.decide()` which either
  shells out to `claude -p --bare --json-schema ...` (LLM mode) or runs the
  rule-based fallback. Both return the same Action JSON.
- **Output** — full sim dumps to `runs/{topology}.json` with `events`,
  `prices_over_time`, `agents`, `comm_matrix`, `summary`. The frontend can
  fetch that file directly.

## Files

```
backend/
├── schemas.py          # Pydantic: AgentState, CommMatrix, Action, Event, Message, RunResult
├── personas.py         # 4 persona templates + 15-buyer profile sequence
├── topology.py         # 5 networkx generators + graph→matrix
├── market.py           # Seller dynamic pricing (yield management, 3 signals)
├── agent_runtime.py    # claude -p invocation + rule-based fallback
├── engine.py           # Tick loop, event emission, world state
├── run.py              # CLI runner
├── app.py              # FastAPI: /api/sim/run, /api/sim/compare, /api/sim/cached
├── requirements.txt
└── README.md
```

## Install

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Quick start

```bash
# Single topology, rule-based (fast)
python run.py --topology small_world --mode rule

# All 5 topologies → runs/*.json (commit these to GitHub for the static demo)
python run.py --all --mode rule

# One topology with real Claude agents (slower; one claude -p per agent per tick)
python run.py --topology isolated --mode claude --model haiku

# Start API + static frontend
uvicorn app:app --port 8000 --reload
# then open http://localhost:8000/demo.html
```

## REST API

- `GET  /api/sim/topologies` — list topologies
- `POST /api/sim/run` — body: `{topology, seed, ticks, mode, model}` → full result
- `GET  /api/sim/cached/{topology}` — serve pre-generated `runs/*.json`
- `GET  /api/sim/compare?seeds=5` — run each topology N times, return mean prices
  and overpay % vs the cheapest topology (the headline "22%" data point)

## Action JSON schema

Every agent decision conforms to:

```json
{
  "action": "BUY" | "COMMUNICATE" | "WAIT",
  "target": "Airline_A" | "C" | null,
  "content": "What price are you seeing?" | null,
  "reasoning": "one-sentence why"
}
```

This is also passed to `claude -p --json-schema` so the LLM is forced to comply.

## Calibrating the 22% spread

The "isolated overpays 22% vs small_world" claim is a function of these knobs
in `agent_runtime.py::_fallback` (and the equivalent strategy hints in the
system prompt for LLM mode):

- `accept_pct` per profile — how aggressive each persona is about pulling the trigger
- `max_wait` per profile — patience ceiling before forced buy
- `Seller.update_price()` coefficients (0.96 / 1.04 / 0.92 / 1.03)

Run `python run.py --all` and check `/api/sim/compare`. Tweak until isolated
≈ 1.22 × small_world.
